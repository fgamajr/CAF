1     Introdução
1.1    Contexto
O Cadastro da Agricultura Familiar (CAF) é um sistema que registra informações sobre propri-
edades rurais de agricultores familiares no Brasil. A base contém aproximadamente 3.392.881
documentos relacionados a imóveis rurais, armazenados no Amazon S3 e referenciados na
tabela S DOCUMENTO do banco de dados PostgreSQL.
   Estes documentos são fundamentais para:

    • Comprovação de elegibilidade para programas governamentais

    • Cálculo de benefı́cios proporcionais à área cultivável

    • Dimensionamento de polı́ticas públicas

    • Planejamento de distribuição de recursos agrı́colas

1.2    Problema Identificado
Durante análises preliminares, identificamos questões crı́ticas sobre a qualidade e conformidade
destes documentos:

    1. Documentos inadequados: Alguns arquivos anexados não são documentos de imóveis
       conforme exigido pela Portaria MDA 19/2025 (notas fiscais, extratos bancários, identida-
       des, etc.)

    2. Ilegibilidade: Documentos com baixa qualidade de digitalização ou danificados

    3. Informação em páginas variadas: A área do imóvel pode aparecer em qualquer página
       do documento, não apenas na primeira

    4. Divergências de área: Quando identificável, a área no documento pode não corresponder
       à área cadastrada no banco de dados

1.3    Objetivo do Trabalho
Realizar auditoria de qualidade em uma amostra aleatória de 201 documentos para:

    • Quantificar tipos de problemas (falsos positivos, ilegibilidade, divergências)

    • Validar correspondência entre área documental e área cadastrada

    • Estimar magnitude dos problemas na população de 3,39 milhões

    • Fornecer evidências documentais completas para cada caso analisado

    • Recomendar ações corretivas baseadas nos achados

2     Metodologia
2.1     Abordagem em Duas Etapas
A análise foi realizada em duas etapas sequenciais:

2.1.1    Etapa 1: Análise Inicial (Outubro 10, 2024)
    • Ferramenta: Claude Vision API (Anthropic)

    • Profundidade: 2 primeiras páginas de cada PDF

    • Objetivo: Identificar se documento é de imóvel e extrair área

    • Resultado: 64 documentos OK (31,8%), 137 problemáticos (68,2%)

2.1.2    Etapa 2: Re-análise Profunda (Outubro 22, 2024 - 02:00)
Dos 137 documentos problem áticos, 136 foram re-analisados com:

    • Profundidade aumentada: Até 10 páginas por PDF

    • Prompt aprimorado: Instruções mais detalhadas

    • Correções de parsing: Tratamento de erros da primeira análise

    • Classificação de tipo: Identificação explı́cita de documentos inadequados

    • Comparação de áreas: Cálculo de divergência percentual vs banco

2.2     Tecnologias Utilizadas

        Componente           Uso
        PostgreSQL           Banco de dados (tabela S DOCUMENTO), host
                             172.23.80.1:5433
        Amazon S3            Armazenamento dos 3,39M PDFs
        Claude Vision API    Análise multimodal de documentos (OCR + compreensão)
        Python 3.11          Scripts de processamento, parsing e comparação
        boto3                SDK AWS para download de arquivos do S3
        pdf2image            Conversão PDF para imagens PNG

2.3     Amostragem
Método: Amostragem Aleatória Simples (AAS)

    • População (N): 3.392.881 documentos

    • Amostra (n): 201 documentos

    • Seleção: Aleatória via ORDER BY RANDOM() no PostgreSQL

    • Nı́vel de confiança: 95%

    • Margem de erro máxima: ±6,44%

5     Análise Agregada de Áreas
5.1    Metodologia
Para verificar se as divergências individuais se acumulam em nı́vel agregado, realizamos uma
análise comparativa entre:

    • Soma das áreas no banco de dados (para os 201 documentos da amostra)

    • Soma das áreas encontradas nos documentos (apenas documentos com área legı́vel)

5.2    Resultados

                    Métrica                                              Valor
                    Documentos analisados                                   201
                    Documentos com área legı́vel                   155 (77,1%)
                    Soma das áreas no BANCO                       1.732,85 ha
                    Soma das áreas nos DOCUMENTOS                 3.827,65 ha
                    Diferença absoluta                            +2.094,80 ha
                    Diferença percentual                           +120,89%

           Tabela 2: Comparação agregada entre áreas do banco e áreas documentais


Nota Metodológica: A comparação agregada foi realizada exclusivamente com os 155 docu-
mentos onde a área foi identificada em ambos os lados (banco de dados e documento digitalizado).
Os 46 documentos restantes (documentos de tipo inadequado, ilegı́veis ou pendentes) foram ex-
cluı́dos da análise para garantir comparabilidade metodológica. As somas de área no banco de
dados referem-se, portanto, aos mesmos 155 documentos utilizados no cálculo da soma das áreas
documentais.

5.3    Interpretação
A análise agregada revela uma divergência significativa de +121% entre as áreas registradas
no banco de dados e as áreas documentadas nos imóveis. Isto significa que, em termos agregados:

    • As áreas nos documentos são 121% maiores que as áreas registradas no banco

    • Dos 155 documentos com área legı́vel, a soma total é 2.095 hectares maior que o regis-
      trado

    • Esta divergência é sistemática, não aleatória

Possı́veis Causas
    1. Erro sistemático no cadastro: Áreas podem ter sido registradas com unidades incor-
       retas (ex: alqueires vs hectares)

    2. Atualizações não refletidas: Documentos mais recentes com áreas maiores, banco de-
       satualizado

    3. Múltiplas propriedades: Documentos podem conter soma de múltiplas propriedades
       enquanto banco registra apenas uma

    4. Diferença metodológica: Área total vs área útil/cultivável

6     Conclusões e Recomendações
6.1    Conclusões
    • Sistema precisa de melhorias significativas na classificação automática de documentos

    • Análise profunda (10 páginas) é essencial - informações aparecem frequentemente após
      página 2

    • Divergências de área são comuns e requerem investigação (erros vs mudanças legı́timas)

    • Base possui qualidade superior ao estimado inicialmente (68% problemas → apenas 5%
      problemas reais)

6.2    Recomendações
    1. Melhorar classificação de tipos: Implementar machine learning para reduzir 33% de
       falsos positivos

    2. Análise profunda como padrão: Processar até 10 páginas automaticamente

    3. Revisar divergências: Investigar 827k documentos com área não correspondente

    4. Estabelecer processo de correção: Remover/reclassificar 1,130k falsos positivos
