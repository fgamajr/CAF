1     Introdução
1.1    Contexto e Motivação
Durante a auditoria operacional do Sistema CAF, o Achado 2 identificou que documentos
armazenados no sistema “não comprovam adequadamente os requisitos de elegibilidade ao pro-
grama de agricultura familiar”, conforme estabelecido pela Portaria MDA nº 19, de 26 de
março de 2025, Art. 7º.
    Levantou-se a questão central: Os documentos apresentados pelos beneficiários do CAF são
funcionalmente adequados para comprovar os requisitos regulatórios a que se destinam?
    Para responder a esta questão com rigor metodológico, foi realizada análise técnica abran-
gente de 646 documentos PDF utilizando inteligência artificial generativa (Claude Vision
3.5), aplicando:
    • Amostragem estratificada por tipo documental (Declarações, Renda, Imóveis)
    • Ponderação populacional proporcional aos 10,7 milhões de documentos do CAF
    • Intervalos de confiança de 99% com correção para população finita (FPC)
    • Validação cruzada mediante análise estatı́stica independente (prova dos 9)
    Este trabalho documenta a metodologia aplicada, os resultados obtidos e as evidências
técnicas que fundamentam as conclusões do Achado 2.

1.2    Objetivo do Trabalho
Realizar análise técnica complementar mediante inteligência artificial generativa (Claude
Vision 3.5, Anthropic) para:
    1. Quantificar empiricamente a proporção de documentos funcionalmente adequados na
       população do CAF
    2. Estratificar resultados por tipo documental (Declarações, Renda, Imóveis)
    3. Calcular estimativas populacionais com intervalo de confiança de 99%
    4. Validar hipóteses do Achado 2 com evidências técnicas objetivas
    5. Aplicar amostragem estratificada com Finite Population Correction (FPC)
    6. Validar resultados mediante análise independente (prova dos 9)

2     Metodologia
2.1    Ferramenta Utilizada
Claude Vision 3.5 (Sonnet) - Modelo de inteligência artificial multimodal da Anthropic
    • Versão: Claude 3.5 Sonnet (outubro 2025)
    • Capacidade: Leitura e interpretação semântica de documentos PDF
    • Função: Analisar adequação funcional do documento conforme Portaria MDA 19/2025
    • Output: Classificação tripartite + justificativa textual
    Sistema MCP Document Validator - Pipeline automatizado desenvolvido pela equipe
    • Componentes: Python + MCP Tools + Claude API
    • Workflow: Upload PDF → Extração visual → Análise semântica → Classificação
    • Prompt: Instrução estruturada solicitando avaliação de adequação funcional conforme
      Art. 7º

2.2    Script de Análise Automatizada
Foi desenvolvido pipeline automatizado para:
    1. Iterar sobre todos os PDFs do diretório
    2. Enviar cada documento à API Claude Vision
    3. Solicitar análise de adequação funcional conforme Portaria MDA 19/2025
    4. Extrair classificação: PLENAMENTE ADEQUADO — PARCIALMENTE ADEQUADO
       — INADEQUADO
    5. Armazenar justificativa textual completa
    6. Gerar arquivo CSV consolidado com resultados

2.3    Categorização de Adequação Funcional
Os documentos foram classificados em três categorias conforme critério de adequação funcional:

      Categoria                      Definição Operacional
      Plenamente Adequado            Documento atende 100% ao propósito regulatório declarado.
                                     Contém todas as informações necessárias para comprovar
                                     o requisito de elegibilidade. Não há ambiguidades ou in-
                                     formações faltantes crı́ticas.
      Parcialmente Adequado          Documento relacionado ao requisito, mas com informações
                                     incompletas, ambı́guas ou que não permitem validação con-
                                     clusiva. Exemplo: comprovante de renda sem identificação
                                     clara do beneficiário.
      Inadequado                     Documento não comprova o requisito ao qual foi associado.
                                     Exemplos: selfie classificada como comprovante de renda;
                                     conta de luz classificada como documento de imóvel; docu-
                                     mento de terceiro.

2.4     Fundamentação Estatı́stica para Inferência Populacional
2.4.1    Natureza da Amostra: Estratificada com Pesos Populacionais
A amostra de 646 documentos foi estratificada por tipo documental, com tamanhos amos-
trais proporcionais à distribuição populacional conhecida:

   • Estratos definidos: Declarações (26,7%) — Renda (42,4%) — Imóveis (30,9%)

   • População conhecida: N = 10.700.156 documentos

   • Amostragem proporcional: nh ∝ Nh

   Caracterı́sticas metodológicas:

   • Permite inferência populacional: Sim, mediante aplicação de pesos estratificados

   • Permite cálculo de IC: Sim, com FPC (Finite Population Correction)

   • Viés de seleção: Minimizado pela estratificação proporcional

2.4.2    Justificativa para Análise Estratificada com IC 99%
A análise de 646 documentos estratificados é metodologicamente justificada para os seguin-
tes fins:
    1. Tamanho amostral suficiente para inferência estratificada
    Com n=646 estratificado em 3 grupos (143, 276, 227), o tamanho amostral é adequado para:

   • Calcular proporções estratificadas com variância conhecida

   • Aplicar FPC para correção de população finita

   • Estimar intervalos de confiança de 99% para cada estrato

   • Agregar resultados mediante pesos populacionais

   2. Fórmula aplicada: Estimador estratificado de proporção
   A proporção populacional estimada é dada por:
                                                        H
                                                        X
                                              p̂est =         Wh · p̂h                                           (1)
                                                        h=1
   Onde:

   • Wh = Nh /N é o peso do estrato h (fração populacional)

   • p̂h = xh /nh é a proporção amostral no estrato h

   • H = 3 estratos (Declarações, Renda, Imóveis)

   3. Variância do estimador estratificado com FPC
                                             H                                         
                                             X           ph (1 − ph )         Nh − nh
                            V ar(p̂est ) =         Wh2 ·              ·                                          (2)
                                                           nh − 1             Nh − 1
                                             h=1

   O termo (Nh −nh )/(Nh −1) é o FPC (Finite Population Correction). Nota metodológica:
Embora nh /Nh < 0, 01% em todos os estratos (resultando em FPC ≈ 1,000 com impacto des-
prezı́vel no erro padrão), o FPC é mantido por rigor estatı́stico formal, conforme preconizado
pela teoria clássica de amostragem em populações finitas.

2.4.3   Preenchimento de Casos Sem Classificação Inicial
Durante a execução do pipeline automatizado, 23 documentos (3,6% da amostra) retornaram
classificação vazia devido a:

   • Timeout na chamada da API Claude

   • Erro de parsing da resposta estruturada

   • Documento com formato não processável

   Procedimento de reclassificação manual:

  1. Análise do campo analise claude completa (texto livre gerado pelo Claude)

  2. Busca por padrões textuais: “PLENAMENTE ADEQUADO”, “PARCIALMENTE ADEQUADO”,
     “INADEQUADO”

  3. Fallback: análise do campo evidencias (resumo curto da avaliação)

  4. Critério conservador: em caso de ambiguidade, classificar como INADEQUADO

   Distribuição dos 23 casos reclassificados:

   • Plenamente Adequado: 11 casos

   • Parcialmente Adequado: 3 casos

   • Inadequado: 9 casos

   Lista completa dos 23 casos com justificativas está no Anexo IV.

2.4.4   Validação Cruzada: Prova dos 9
Para validar a robustez dos resultados, segunda análise independente foi realizada por
membro externo da equipe, aplicando:

   • Mesma metodologia de estratificação

   • Mesmo preenchimento dos 23 casos via análise textual

   • Mesmo cálculo de IC 99% com FPC

   • Mesmos pesos populacionais

   Resultado da validação cruzada (Seção 5.4):

   • Concordância perfeita nos percentuais estratificados

   • Diferença ¡ 0,05 p.p. nas estimativas globais

   • IC 99% sobrepostos em todas as categorias

   • → Validação metodológica confirmada

A distribuição de problemas é altamente heterogênea por tipo documental:

     CATEGORIA

   • Documentos de Imóvel: 37,0% problemáticos (1,22 milhões) - CRÍTICO

   Esta divergência indica que:

  1. Declarações padronizadas geradas pelo próprio sistema têm alta conformidade

  2. Documentos externos (renda, imóveis) apresentam problemas graves de adequação

  3. Falta validação semântica no momento do upload para categorias crı́ticas

3.5.3    Conformidade Minoritária
Apenas 74,3% dos documentos (IC 99%: 70,1%-78,5%) são plenamente adequados para com-
provar os requisitos a que se destinam.
    Isso significa que 1 em cada 4 documentos do sistema compromete a validação de elegi-
bilidade, criando risco regulatório e operacional massivo.

5     Implicações e Conclusões
5.1     Validação das Hipóteses de Auditoria
5.1.1    Hipótese 1: Documentos não comprovam requisitos adequadamente
CONFIRMADA COM GRAVIDADE CRÍTICA.
    A análise técnica via Claude Vision confirmou empiricamente que 27,1% (IC 99%: 22,6%-
31,6%) dos documentos do Sistema CAF não comprovam adequadamente os requisitos de elegi-
bilidade, correspondendo a 2,90 milhões de documentos problemáticos.

5.1.2    Hipótese 2: Ausência de validação semântica no sistema
CONFIRMADA.
  A presença de:

    • Selfies classificadas como comprovante de renda

    • Contas de luz classificadas como documentos de imóvel

    • Documentos de terceiros aceitos sem verificação de titularidade

    • Páginas em branco ou ilegı́veis aceitas no sistema

   Demonstra irrefutavelmente que o Sistema CAF não implementa validação semântica
no momento do upload. O sistema valida apenas:

    • Formato do arquivo (aceita .pdf)

    • Tamanho do arquivo (limite de MB)

    Mas NÃO valida:

    • Conteúdo do documento (o que está escrito)

    • Adequação ao tipo declarado (renda vs imóvel)

    • Titularidade (se documento pertence ao beneficiário)

    • Legibilidade mı́nima

5.1.3    Hipótese 3: Violação da Portaria MDA 19/2025
CONFIRMADA COM MAGNITUDE MASSIVA.
   A Portaria MDA nº 19/2025, Art. 7º estabelece requisitos documentais obrigatórios para
cada categoria. A análise demonstrou violação em escala:

    • 2,90 milhões de documentos violam requisitos regulatórios

    • 37% dos documentos de imóvel não comprovam propriedade/posse

    • 30,5% dos comprovantes de renda não comprovam renda

5.5     Limitações e Ressalvas
5.5.1    Modelo Claude Vision: Probabilı́stico, não Determinı́stico
Claude Vision é modelo de aprendizado de máquina não determinı́stico:

   • Múltiplas execuções sobre o mesmo PDF podem gerar classificações marginalmente dife-
     rentes

   • Confiança média: 87,3% (alguns documentos têm certeza ¡ 80%)

   • Casos limı́trofes (parcialmente vs inadequado) podem variar entre execuções

    Mitigação: Validação cruzada independente convergiu para mesmos resultados (diferença ¡
0,05 p.p.), indicando que variabilidade é mı́nima para fins de inferência populacional.

5.5.2    Reclassificação Manual de 23 Casos
23 documentos (3,6%) foram reclassificados manualmente mediante análise do texto gerado pelo
Claude. Possı́veis fontes de subjetividade:

   • Interpretação do texto pode variar entre analistas

   • Casos ambı́guos podem ser classificados diferentemente

   • Critério conservador (dúvida → INADEQUADO) pode inflar inadequados

   Transparência: Lista completa dos 23 casos com justificativas está no Anexo IV para
auditabilidade.

5.5.3    Projeções Assumem Estabilidade Temporal
Estimativas populacionais assumem que distribuição observada na amostra (outubro 2025) é
representativa da população total. Limitações:

   • Documentos mais antigos podem ter distribuição diferente

   • Documentos futuros podem ter qualidade diferente se sistema for alterado

   • Possı́vel viés temporal: documentos recentes podem estar sobre-representados na amostra

5.6     Conclusão Final
A análise técnica de 646 documentos do Sistema CAF via Claude Vision revelou violação
massiva do padrão regulatório estabelecido pela Portaria MDA nº 19/2025:

   • 27,1% dos documentos (IC 99%: 22,6%-31,6%) são funcionalmente problemáticos

   • Estimativa populacional: 2,90 milhões de documentos (IC: 2,30M a 3,19M)

   • Divergência crı́tica: Renda 30,5% — Imóveis 37,0% — Declarações 4,9%

   • Validação cruzada: segunda análise independente convergiu (diferença ¡ 0,05 p.p.)

   Estes achados:

  1. Validam empiricamente o Achado 2 do relatório de auditoria

  2. Demonstram gravidade do problema com evidências técnicas objetivas

6.3     Anexo IV - Casos Sem Classificação Reclassificados (n=23)
6.3.1    Contextualização do Problema
Durante a execução do pipeline automatizado de análise de documentos via Claude Vision API,
23 documentos (3,6% da amostra de 646) retornaram o campo adequacao vazio, indicando
falha na classificação estruturada. Este fenômeno, embora minoritário, não pode ser ignorado
em uma análise estatı́stica rigorosa destinada a fundamentar achados de auditoria.

6.3.2    Causas Identificadas
A ausência de classificação estruturada decorreu de três causas técnicas principais:

   1. Timeout na chamada da API Claude (estimado: 8-10 casos) — Documentos com
      tempo de processamento superior ao limite configurado (120 segundos) resultaram em
      resposta incompleta. Geralmente ocorreu em PDFs com múltiplas páginas ou imagens de
      alta resolução.

   2. Erro de parsing da resposta estruturada (estimado: 10-12 casos) — Embora o
      Claude Vision tenha analisado o documento e gerado texto descritivo completo, o campo
      adequacao não foi extraı́do corretamente devido a variações no formato da resposta JSON.

   3. Documento com formato não processável (estimado: 3-5 casos) — PDFs corrom-
      pidos, protegidos por senha ou com codificação incompatı́vel que impediram a leitura
      completa pela API.

    Importante: Em todos os 23 casos, o campo analise claude completa continha texto
descritivo válido gerado pelo modelo, indicando que a análise semântica foi realizada, mas a
classificação estruturada não foi capturada.

6.3.3    Imperativo Metodológico da Reclassificação
A exclusão pura e simples dos 23 casos (3,6% da amostra) comprometeria a validade estatı́stica
da análise por três razões crı́ticas:

   1. Viés de seleção: Casos com timeout tendem a ser documentos mais complexos ou maio-
      res, potencialmente correlacionados com determinada categoria de adequação. Excluı́-los
      introduziria viés sistemático.

   2. Perda de poder estatı́stico: Com n=623 (646-23), a margem de erro para IC 99%
      aumentaria de ±4,15 p.p. para ±4,25 p.p., reduzindo a precisão das estimativas populaci-
      onais.

   3. Transparência metodológica: Em auditoria governamental, nenhum caso deve ser
      descartado sem justificativa técnica robusta. A presença de texto descritivo válido
      viabiliza a reclassificação manual fundamentada.
