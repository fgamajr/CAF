1 Introdução
1.1 Contexto e Motivação
Durante a elaboração do Achado 4 ("Incompletude e baixa qualidade do Dicionário de Dados
do Sistema CAF"), identificou-se uma aparente divergência entre o número de tabelas existentes
no banco de dados e o número de tabelas documentadas no artefato “Dicionario de dados-
CAF3.xlsx”. A versão inicial da Seção II do achado apontava um déficit de 16 tabelas, com base
em informações preliminares.
Este papel de trabalho foi criado para formalizar a apuração dessa divergência, estabelecendo
uma evidência técnica, rastreável e reprodutível sobre a real cobertura do dicionário de dados.
Questão de Auditoria Central
A documentação oficial do Sistema CAF (dicionário de dados) reflete com precisão o
esquema de tabelas existente no banco de dados de produção?
1.2 Objetivos do Papel de Trabalho
1.Objetivo Primário : Identificar e listar de forma conclusiva todas as tabelas do schema
caf_mapa que não estão documentadas no dicionário de dados oficial.
2.Objetivo Secundário : Quantificar o déficit exato de cobertura da documentação.
3.Objetivo Terciário : Gerar uma tabela comparativa completa que sirva como evidência
para o Achado 4, indicando o status de documentação de cada tabela do banco de dados.
2 Metodologia
2.1 População e Amostra
A análise foi **censitária**, comparando duas populações completas:
•População 1 (Banco de Dados) : A lista de todas as tabelas base ( BASE TABLE ) no
schemacaf_mapa do banco de dados PostgreSQL de produção.
•População 2 (Dicionário) : A lista de todas as tabelas documentadas, extraída do
arquivoDicionario de dados-CAF3v1.CSV .
2.2 Procedimentos Analíticos
A análise foi realizada em três etapas, utilizando scripts bashe o cliente psql. Os scripts
completos estão no Anexo C.
1.Extração da Lista do Banco de Dados : Uma consulta SQL foi executada no banco
de dados para extrair os nomes de todas as tabelas do tipo BASE TABLE , excluindo VIEWs.
A saída foi limpa e salva no arquivo db_tables.txt .
2.Extração da Lista do Dicionário : O arquivo Dicionario de dados-CAF3v1.CSV foi
processado para extrair da primeira coluna os nomes de tabelas que seguem o padrão
S_.... A saída foi limpa, ordenada e salva no arquivo dict_tables.txt .
3.Comparação Diferencial : O comando commdo Linux foi utilizado para comparar os
dois arquivos e gerar três listas distintas:
•Tabelas presentes apenas no banco de dados (não documentadas).

3 Resultados
3.1 Quantificação do Déficit de Cobertura
A análise confirmou a existência de **109 tabelas base** no schema caf_mapa . O dicionário de
dados, por sua vez, documenta **93 tabelas**.
Isso resulta em um déficit de **16 tabelas não documentadas**, o que corresponde a uma
cobertura de apenas 85,3% e um déficit de **14,7%**.

4 Conclusão
Este papel de trabalho confirma, de maneira conclusiva e reprodutível, que o dicionário de dados
do Sistema CAF possui um déficit de cobertura de **14,7%**. Das 109 tabelas base existentes
no banco de dados, 16 não estão documentadas.
Esta evidência técnica fundamenta o Achado 4 e deve ser utilizada para corrigir a Seção II
do relatório de auditoria, garantindo a precisão das informações apresentadas.

B Anexo C – Scripts Utilizados
B.1 Script de Análise e Comparação
Listing 1: Script para identificar tabelas não documentadas
1#!/bin/bash
2
3# --- STEP 1: Get a clean list of BASE TABLES from the database ---
4DB_TABLES_FILE="db_base_tables.txt"
5echo"Executando␣query␣no␣banco␣de␣dados..."
6PGPASSWORD=’[SENHA_REMOVIDA]’ psql -h 172.23.80.1 -p 5433 -U postgres -d caf_mapa -t -
c "SELECT␣table_name␣FROM␣information_schema.tables␣WHERE␣table_schema␣=␣’caf_mapa
’␣AND␣table_type␣=␣’BASE␣TABLE’␣ORDER␣BY␣table_name;" | grep -v "rows)" | sed’s/␣
//g’ | grep . > "$DB_TABLES_FILE"
7
8# --- STEP 2: Get a clean list of tables from the dictionary CSV ---
9DICT_CSV_PATH="Dicionario␣de␣dados-CAF3v1.CSV"
10DICT_TABLES_FILE="dict_tables.txt"
11echo"Processando␣arquivo␣CSV␣do␣dicionário..."
12cat"$DICT_CSV_PATH" | tr -d ’\r’ | cut -d, -f1 | sed’s/␣//g’ | grep -E ’^S_[A-Z_]+$’
| sort -u > "$DICT_TABLES_FILE"
13
14# --- STEP 3: Compare the two clean lists ---
15echo"Tabelas␣não␣documentadas:"
16comm -23 "$DB_TABLES_FILE" "$DICT_TABLES_FILE"
